# 📄 서비스 정책 (Service Policy)

## 🎯 서비스 개요

농산물 특가 판매 이커머스 플랫폼으로,  
상품 조회부터 장바구니, 주문, 쿠폰, 검색 기능을 제공합니다.

---

## 🔑 핵심 정책 요약

- Redis 분산 락 기반 선착순 쿠폰 발급
- Redis 분산 락 + DB 조건부 UPDATE 기반 상품 재고 차감
- 특가 상품 포함 시 쿠폰 사용 불가
- 최소 주문 금액 20,000원
- 검색 API 캐시 적용 (v1 / v2 비교 구조)

---

## 🎯 핵심 목표

- **데이터 정합성 보장**: 동시성 제어를 통한 재고 및 쿠폰 관리
- **성능 최적화**: 캐싱 전략을 통한 검색 성능 개선
- **실제 서비스 환경 고려**: 특가 상품, 쿠폰, 재고 등의 복합 조건을 반영한 비즈니스 정책 설계

---

## 👤 회원 및 인증 정책

### 회원가입 및 관리

- 이메일: 필수, 유니크
- 비밀번호: 영문 + 숫자 포함 8~12자리
- 전화번호: 010 포함 11자리

### 제한 사항

- 이메일 / 비밀번호 / 권한 변경 불가
- 회원 탈퇴 미구현

### 인증 방식

- JWT 기반 인증
- 만료 시간: 60분

---

## 🥬 상품 및 타임세일 정책

### 상품 유형 및 상태

- 유형: NORMAL / SPECIAL
- 상태: READY / ON_SALE / SALE_ENDED / SOLD_OUT / HIDDEN

### 가격 정책

- normalPrice / salePrice / specialPrice
- 타임세일 기간에는 `specialPrice` 우선 적용

### 타임세일

- Quartz 스케줄러 기반 자동 실행
- 시작/종료 시점에 상태 자동 변경
- 재고 0 시 SOLD_OUT 처리

---

## 🛒 장바구니 및 주문 정책

### 장바구니

- 로그인 사용자만 사용 가능
- 동일 상품 추가 시 수량 증가
- 품절 / 판매 종료 상품 담기 불가
- 조회 시점에 상품 상태 실시간 반영
- 장바구니는 재고 예약 기능이 아니며, 실제 재고 차감은 주문 시점에 수행

### 주문

- 장바구니 전체 주문
- 최소 주문 금액 20,000원
- 주문 성공 시:
    - 재고 차감
    - 쿠폰 사용 처리
    - 장바구니 초기화

### 제한

- 주문 취소 및 결제 시스템 미구현

---

## 🎟 쿠폰 정책 (선착순)

### 발급 조건

- 사용자당 쿠폰 1개당 1회만 발급 가능
- 수량 제한 및 기간 제한
- 발급 후 5일 유효

### 동시성 제어

- 선착순 쿠폰 발급은 Redis 분산 락 기반으로 제어
- 동일 쿠폰에 대한 동시 발급 요청을 제한
- 락을 획득한 요청만 발급 로직 수행
- 락 획득 실패 시 즉시 실패 처리 (Fail Fast)

### 사용 조건

- 최소 주문 금액 20,000원 이상
- 일반 상품만 포함된 주문에서 사용 가능
- 특가 상품 포함 시 사용 불가
- 1주문 1쿠폰

### 할인 방식

- 정액 할인

### 상태

- AVAILABLE: 사용 가능
- USED: 사용 완료
- EXPIRED: 만료

---

## ⚡ 기술 특화 정책

### 동시성 제어 대상

- 선착순 쿠폰 발급
- 상품 주문 시 재고 차감

---

### 1️⃣ 선착순 쿠폰 발급 동시성 제어

- Redis 분산 락을 사용하여 쿠폰 발급 동시성 제어
- 쿠폰 ID 기준으로 락을 생성하여 동일 쿠폰에 대한 요청을 직렬화
- 중복 발급 및 초과 발급 방지

#### 처리 흐름

1. 쿠폰 발급 요청
2. 쿠폰 ID 기준 Redis 락 획득
3. 발급 가능 시간 검증
4. 중복 발급 여부 확인
5. DB 조건부 UPDATE로 잔여 수량 검증 및 발급 수량 증가
6. UserCoupon 생성
7. 락 해제

---

### 2️⃣ 상품 재고 차감 동시성 제어

주문 생성에는 Redis 분산 락이 적용되며,  
재고 차감은 DB 조건 검증을 통해 최종 보장됩니다.

즉,  
**Redis 락으로 요청 진입을 제어하고, DB 조건 검증으로 재고 음수를 방지하는 이중 방어 구조**입니다.

#### 처리 전략

- Redis 락으로 동일 상품에 대한 동시 주문 요청 제한
- 재고는 주문 시점에서만 차감
- 재고 부족 시 주문 실패 처리
- 여러 상품 주문 시 상품 ID를 정렬하여 락 획득 순서를 통일
- Deadlock 방지를 위한 설계 적용

#### 처리 흐름

1. 주문 요청
2. 상품 ID 기준 Redis 락 획득
3. 주문 생성 로직 수행
4. 재고 검증 및 차감
5. 주문 저장 및 쿠폰 사용 처리
6. 장바구니 비우기
7. Redis 락 해제

---

### ⚠️ 설계 의도

- Redis 락으로 동시 요청 경쟁 완화
- DB 조건 검증으로 최종 데이터 정합성 보장
- 성능과 안정성을 동시에 확보하기 위한 구조

## 🔍 검색 및 캐싱

### 검색 API

- v1: 캐시 미적용
- v2: Caffeine 캐시 적용

### 인기 검색어

- Redis Sorted Set 기반
- 검색 발생 시점 실시간 집계
- 사용자별 동일 키워드 1회만 카운팅
- 매일 자정 초기화
- 일별 누적 기준 TOP 10 조회

### 캐시 정책

- TTL 5분
- 최대 캐시 수 100개
- 로컬 캐시 사용
