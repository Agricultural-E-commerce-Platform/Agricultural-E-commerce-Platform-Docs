# 🛠 서버 간 DB / Redis 연결 실패 | 2026.04.16

### ❗ 문제 상황

- 서버1(Spring Boot)에서 서버3(MySQL, Redis) 연결 실패
- `nc -zv` 명령어로 외부 IP(125.181.213.48) 포트 연결 시 `Operation timed out` 발생
- HikariCP에서 DB 연결 시 무한 대기 상태 발생

---

### 🔍 원인

- 서버1과 서버3이 동일한 외부 IP를 공유하지만, 서로 다른 Docker 컨테이너로 실행 중이었음
- 외부 IP를 통한 접근 시 포트가 열려 있지 않아 통신 불가
- 두 컨테이너는 동일한 Docker 내부 네트워크(172.18.0.x)에 속해 있었음

---

### 🛠 해결 방법

- `ip addr` 명령어로 각 컨테이너의 내부 IP 확인
- `.env` 파일의 DB 및 Redis 호스트를 외부 IP → 내부 IP로 변경

```env
DB_HOST=172.18.0.4
REDIS_HOST=172.18.0.4
DB_PORT=13306
REDIS_PORT=16379
```

### ✅ 결과
- 내부 IP로 변경 후 HikariCP DB 연결 성공
- Spring Boot 애플리케이션 정상 실행 확인
- MySQL Workbench에서 테이블 생성 확인

### 📌 배운 점
- 동일한 물리 서버의 컨테이너 간 통신은 외부 IP가 아닌 내부 네트워크를 사용해야 한다
- Docker 환경에서는 네트워크 구조를 먼저 이해하는 것이 중요하다
- ip addr를 통해 컨테이너 내부 IP를 확인할 수 있다
- 외부 IP가 아닌 내부 네트워크 대역(172.18.0.x)으로 통신하면 포트 제한 없이 접근 가능하다
