# 🛠 v2 검색 API 인기검색어 미카운팅 문제 (Optional Authentication 적용) | 2026.04.23

### ❗ 문제 상황

- 로그인 상태에서 v2 검색 API 호출 시 인기검색어가 카운팅되지 않음
- Redis에 인기검색어 키(날짜 기반)가 생성되지 않음

---

### 🔍 원인

- `JwtFilter`의 화이트리스트에 `/api/v2/products/**`가 포함되어 있어 토큰 검증이 수행되지 않음
- 인증 과정이 생략되면서 `AuthUser`가 `SecurityContext`에 저장되지 않음
- 컨트롤러에서 `authUser`가 항상 `null`로 처리되어 인기검색어 카운팅 로직이 실행되지 않음

---

### 🛠 해결 방법

- `/api/v2/products/**`를 화이트리스트에서 제거

- 해당 API에 **선택적 인증(Optional Authentication)** 방식 적용

    - 토큰 없음 → 비로그인 사용자로 검색 허용
    - 토큰 유효 → `AuthUser` 주입 후 인기검색어 카운팅
    - 토큰 만료/오류 → 예외를 무시하고 비로그인 사용자로 처리

---

### ✅ 결과

- 비로그인 사용자도 v2 검색 API 정상 사용 가능
- 로그인 사용자는 검색 시 Redis에 인기검색어가 정상적으로 카운팅됨

---

### 📌 배운 점

- 화이트리스트는 인증을 완전히 우회하기 때문에 로그인 여부에 따라 동작이 달라지는 API에는 적합하지 않다
- Optional Authentication 패턴을 사용하면 비로그인 사용자 접근을 허용하면서도, 로그인 사용자에게 추가 기능을 제공할 수 있다
- 인증 처리 방식은 단순 보안뿐 아니라 **비즈니스 로직과도 밀접하게 연결되어 있다**
