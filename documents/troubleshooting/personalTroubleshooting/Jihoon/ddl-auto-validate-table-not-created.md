# 🛠 DB 테이블 생성 실패 (ddl-auto 설정 문제) | 2026.04.27


### ❗ 문제 상황

- AWS RDS 연결 후 회원가입 요청 시 500 에러 발생
- DB에 접속해보니 테이블이 생성되지 않은 상태
- 애플리케이션은 정상 실행되지만 데이터 저장 실패

---

### 🔍 원인

- 운영 환경에서 `ddl-auto: validate` 설정 사용
- `validate`는 테이블을 생성하지 않고 스키마 검증만 수행
- 초기 배포 단계에서 필요한 테이블이 생성되지 않음

---

### 🛠 해결 방법

- 초기 배포 시 `ddl-auto: update`로 변경하여 테이블 자동 생성
- 테이블 생성 및 정상 동작 확인 후 `validate`로 다시 변경
- 운영 환경에서는 스키마 변경 방지를 위해 `validate` 유지

```yaml
# 초기 배포
spring:
  jpa:
    hibernate:
      ddl-auto: update

# 안정화 이후
spring:
  jpa:
    hibernate:
      ddl-auto: validate
```

### ✅ 결과
- 테이블 정상 생성 확인
- 회원가입 및 DB 연동 기능 정상 동작
- 운영 환경에서도 안정적인 스키마 관리 가능
### 📌 배운 점
- update와 validate의 역할 차이를 명확히 이해
- 운영 환경에서는 자동 스키마 변경이 위험할 수 있음
- 초기 배포 단계와 운영 단계에서 설정 전략을 다르게 가져가야 함
