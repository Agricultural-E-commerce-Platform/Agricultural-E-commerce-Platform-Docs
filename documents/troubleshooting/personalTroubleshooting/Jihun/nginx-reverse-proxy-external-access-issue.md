# 🛠 외부에서 Spring Boot 앱 접근 불가 (Nginx Reverse Proxy 설정) | 2026.04.16

### ❗ 문제 상황

- 서버1(Spring Boot)이 정상 실행 중이나 외부에서 접근 불가
- `curl http://125.181.213.48:8080` 요청 시 연결 실패
- Postman을 통한 API 테스트 불가

---

### 🔍 원인

- 서버1 컨테이너의 8080 포트가 외부에 직접 노출되지 않은 구조
- 하나의 물리 서버에 여러 Docker 컨테이너가 실행되는 환경
- 외부 요청을 내부 컨테이너로 전달할 **리버스 프록시(Nginx)** 설정이 없었음

---

### 🛠 해결 방법

- 서버3의 Nginx를 통해 외부 요청을 수신하도록 구성
- 외부 포트(18080) → 내부 컨테이너(Spring Boot 8080)로 포워딩 설정

```nginx
server {
    listen 18080;

    location / {
        proxy_pass http://172.18.0.2:8080;
    }
}
```
- 내부 IP 기반으로 컨테이너 간 통신하도록 구성
- Nginx 리버스 프록시 설정 적용

### ✅ 결과
- 외부에서 http://125.181.213.48:18080로 Spring Boot API 접근 가능
- Postman을 통한 API 테스트 정상 수행
### 📌 배운 점
- Docker 환경에서는 컨테이너 포트가 자동으로 외부에 노출되지 않는다
- 외부 접근을 위해서는 포트 매핑 또는 리버스 프록시 설정이 필요하다
- Nginx를 활용하면 외부 요청을 내부 컨테이너로 안정적으로 라우팅할 수 있다
- 컨테이너 간 통신은 내부 네트워크(IP 기반)를 사용하는 것이 기본 구조이다
