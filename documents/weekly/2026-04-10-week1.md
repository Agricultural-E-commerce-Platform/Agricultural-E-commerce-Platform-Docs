## ✅ 논의 내용 및 결정 사항
- ## 📌 이번 주 합의한 비즈니스 정책 정리
### 상품
- 상품 유형: NORMAL(일반) / SPECIAL(특가)
- 상품 상태: ON_SALE(판매중) / SALE_ENDED(판매종료) / SOLD_OUT(품절)
- 타임세일 종료 시 상품 상태 → SALE_ENDED (스케줄러 처리)
- 하나의 상품은 하나의 타임세일만 가능 (1:1 관계, UNIQUE 제약)

### 주문/장바구니
- 장바구니와 주문 도메인 분리 (CART / ORDER 테이블 별도)
- 장바구니: 상품 담기 → 수량 변경 → 상품 삭제 (소프트 딜리트)
- 주문 흐름: 장바구니 → 구매하기(PENDING) → 결제하기(COMPLETED)
- 최소 주문 금액: 20,000원 이상
- 배송비: 무료
- 주문 취소 버튼 없음 (재고 부족 등 실패 시 자동 CANCELLED 처리)

### 쿠폰
- 할인 방식: 할인 금액으로 통일 (할인율 X)
- 일반 상품: 20,000원 이상일 때 쿠폰 사용 가능
- 특가 상품: 쿠폰 사용 불가
- 혼합 주문 (일반 + 특가): 쿠폰 사용 불가
- 특가 상품: 20,000원 이상일 때 구매 가능
- 쿠폰 수량: total_count만 관리, 0이 되면 발급 종료
- 중복 발급 불가 (user_id + coupon_id 복합 유니크 제약)
- 1주문당 1쿠폰만 사용 가능

### 인기 검색어
- 10분 단위 업데이트
- 동일 사용자 동일 키워드 1회만 카운팅
- 매일 00시 초기화
- Top 10 조회

### 동시성 제어
- Redis 분산락(Lettuce) 으로 통일
- 적용 대상: 타임세일 구매, 선착순 쿠폰 발급, 주문 재고 차감
